End-of-session debrief. Capture everything needed for the next session to pick up cleanly.

Do the following steps in order:

1. **Read all project docs**: Read TODO.md, CHANGELOG.md, DECISIONS.md, SCRATCHPAD.md, ARCHITECTURE.md, and LESSONS.md to understand their current state.

2. **Summarize the session**: What was accomplished? What was attempted but not finished? Were there any surprises or blockers?

3. **Update TODO.md**: Check off completed items. Add new items for unfinished work or follow-ups discovered during the session.

4. **Update CHANGELOG.md**: Add entries under [Unreleased] for everything that changed.

5. **Update DECISIONS.md**: Log any decisions made during the session — even small ones (e.g., "chose X library over Y", "decided to inline styles instead of extracting", "deferred feature Z").

6. **Update SCRATCHPAD.md**:
   - Refresh "Current Status" with exactly where things stand
   - Add a "Session Notes — [today's date]" section at the bottom with:
     - What's done
     - What's in progress / partially complete
     - Blockers or open questions for next session
     - Any context the next session will need that isn't obvious from the code

7. **Update ARCHITECTURE.md**: If any structural changes were made (new files, moved components, changed state shape, new dependencies), update diagrams and tables.

8. **Update LESSONS.md**: Add any new patterns, gotchas, or debugging insights from this session.

9. **Print a handoff summary**: A concise block the user (or next session's Claude) can read to immediately understand where to resume.

10. **Ask about committing**: Offer to commit all doc updates with a descriptive message.
